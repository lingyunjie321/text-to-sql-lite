# 运行时配置实施记录

本文档原本是运行时数据库与模型路由配置的实施计划。当前能力已经落地，因此本文档更新为实施记录和现状对照；判断当前行为时，以源码、测试、[运行时数据库与模型路由配置方案](运行时数据库与模型路由配置方案.md) 和 README 为准。

## 当前状态

运行时配置已支持：

1. 前端创建短生命周期运行配置。
2. 数据库可选择 `workflow.yaml` 中的预设连接，也可提交自定义 SQLite/PostgreSQL/MySQL 连接。
3. `light` 和 `strong` 两档模型必须同时配置，可以使用预设或自定义 OpenAI-compatible / Mock provider。
4. 创建配置时统一测试数据库连接和模型连通性。
5. 创建成功后返回 `runtime_config_id`，后续 `/api/v1/query`、`/api/v1/schema` 和 `/api/v1/sql/execute` 都可以使用该配置。
6. 不传 `runtime_config_id` 时继续使用默认 `workflow.yaml` 配置。
7. 用户提交的数据库密码和模型 API Key 只保存在后端内存运行时会话中，不写入文件，也不回传给前端。

- `frontend/src/App.tsx`：运行配置侧边面板和运行配置状态。
- `frontend/src/App.test.tsx`：运行配置交互测试。

## 当前接口

```http
GET /api/v1/runtime/options
POST /api/v1/runtime/configs
POST /api/v1/query
GET /api/v1/schema?runtime_config_id=rt_xxx
POST /api/v1/sql/execute
```

当前没有独立的 `/api/v1/runtime/database/test` 或 `/api/v1/runtime/models/test`。连通性检查发生在 `POST /api/v1/runtime/configs` 创建配置时。

## 与历史计划的差异

| 历史计划 | 当前实现 |
| --- | --- |
| 独立 `RuntimeConfigPanel.tsx` 组件 | 运行配置面板内联在 `frontend/src/App.tsx`。 |
| 独立数据库测试和模型测试接口 | 创建运行时配置时统一测试。 |
| 将配置摘要写入 `sessionStorage` | 当前只保存在 React state，不持久化敏感信息或配置摘要。 |
| `unsupported_runtime_provider` 错误码 | 当前为 `runtime_provider_unsupported`。 |
| `model_route_incomplete` 错误码 | 当前由 Pydantic/FastAPI 请求校验返回 `validation_error`。 |

## 后续可选优化

- 如果需要更好的前端可维护性，可以把 `RuntimeConfigPanel` 从 `App.tsx` 拆成独立组件，并同步补组件测试。
- 如果需要更细粒度体验，可以新增独立“测试数据库连接”和“测试模型连接”接口，但必须同步更新 API client、集成测试和本文档。
- 如果运行时配置需要跨服务实例共享，再评估持久化密钥管理或外部存储；当前项目范围内不引入 Redis、认证、多租户或长期密钥托管。
