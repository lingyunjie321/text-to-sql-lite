## 核心模块职责表

| 模块 | 入口文件 | 一句话职责 | 关键依赖 |
| --- | --- | --- | --- |
| FastAPI app | `src/text_to_sql_demo/main.py` | 注册健康检查、查询、运行记录、schema、只读 SQL 执行和方言转换接口。 | FastAPI、`TextToSQLApiService` |
| API service | `src/text_to_sql_demo/api/service.py` | 连接 API 与 workflow，负责配置、runtime config 解析、数据库初始化、schema 读取、engine 创建和响应序列化。 | `WorkflowEngine`、`NodeFactory`、`RuntimeConfigResolver`、`LLMClient` |
| Runtime config | `src/text_to_sql_demo/runtime/` | 保存前端创建的短生命周期运行配置，自动发现本地 SQLite 预设，并解析为数据库 URL、目标方言、LLM client 和模型 profiles。 | `RuntimeConfigStore`、`RuntimeConfigResolver`、`discover_sqlite_databases` |
| Metadata store | `src/text_to_sql_demo/metadata/` | 用项目内部 SQLite 保存运行记录、节点 Trace、收藏 SQL 和用户反馈，不写入业务目标库。 | SQLAlchemy、Pydantic |
| Memory helpers | `src/text_to_sql_demo/memory/` | 把 approved saved_query 转为可信 Reference SQL，并合并 YAML knowledge 命中。 | `MetadataStore`、`KnowledgeStore` |
| Workflow core | `src/text_to_sql_demo/workflow/engine.py` | 按配置执行节点并记录 Trace。 | `WorkflowConfig`、`NodeFactory`、`NodeResult` |
| Node registry/factory | `src/text_to_sql_demo/workflow/registry.py`、`factory.py` | 通过注册表按 node type 创建节点，避免 engine 硬编码具体节点。 | `BaseNode`、`NodeConfig` |
| Nodes | `src/text_to_sql_demo/nodes/__init__.py` | 导入节点模块触发 `@register_node` 注册。 | `BeginNode`、`SelectionNode`、`ContextRetrievalNode` 等节点类 |
| Schema linking | `src/text_to_sql_demo/schema/linking.py` | 用表/列名称、中文同义词、描述和外键扩展选出相关 schema 子集。 | `DatabaseSchemaMetadata` |
| Example retrieval | `src/text_to_sql_demo/retrieval/examples.py` | 从 `configs/examples.yaml` 加载示例并按词法相似度和表重叠返回 Top-K。 | PyYAML、Pydantic |
| Knowledge retrieval | `src/text_to_sql_demo/retrieval/knowledge.py` | 从 `configs/knowledge.yaml` 加载 Reference SQL、文档、Metric、Semantic Model，并按问题和 linked tables 返回 Top-K。 | PyYAML、Pydantic |
| Business patterns | `src/text_to_sql_demo/retrieval/patterns.py` | 从 `configs/dialect_patterns.yaml` 加载业务方言范式，并按问题、方言和表重叠返回 Top-K。 | PyYAML、Pydantic |
| Reflection policy | `src/text_to_sql_demo/reflection/` | 把 SQL 错误映射为 FIX_SQL、RELINK_SCHEMA、REASONING_REWRITE、HITL 等策略，并生成成功/失败 SQLContext 脱敏摘要。 | Pydantic、`SQLError` |
| SQL generation | `src/text_to_sql_demo/nodes/sql_generation.py` | 在 `GenSQLAgenticNode` 内完成复杂度分类、模型路由、RAG 上下文/业务范式注入、prompt 构建和 LLM 调用。 | `PromptBuilder`、`ComplexityClassifier`、`LLMClient` |
| SQL validation | `src/text_to_sql_demo/sql/validator.py` | 用 SQLGlot 做方言解析、单语句、只读 SELECT、schema 引用和可选方言渲染。 | SQLGlot、schema metadata |
| SQL execution | `src/text_to_sql_demo/execution/sql_executor.py` | 用 SQLAlchemy 执行已校验 SQL，并归一化 columns/rows/duration；执行节点要求校验方言与执行方言一致。 | SQLAlchemy |
| Frontend app | `frontend/src/App.tsx` | 提供自然语言查询、运行配置、SQL 编辑执行、结果表格、修复提示和开发者信息。 | React、API client |

## 配置与数据文件

- `workflow.yaml`：真实工作流配置，声明 `nodes` 和 `edges`，模型 alias 为 `light`、`strong`。
- `configs/examples.yaml`：当前 `ExampleRetrievalNode` 默认读取的历史 SQL 示例。
- `configs/knowledge.yaml`：当前 `ContextRetrievalNode` 默认读取的本地知识库 fallback。
- `configs/dialect_patterns.yaml`：当前 `GenSQLAgenticNode` 默认读取的业务 SQL 方言范式。
- `configs/prompts/`：生成和修复节点默认使用的 YAML prompt 模板。
- `.env.example`：包含 `DEMO_DATABASE_URL`、`TEXT_TO_SQL_WORKFLOW_CONFIG`、`DEEPSEEK_API_KEY`、`DEEPSEEK_BASE_URL` 等本地环境变量示例。
- `data/sqlite/demo.db`：可由 `python scripts/init_db.py --db-path data/sqlite/demo.db` 重新生成。
- `examples/minimal_workflow.yaml`：用于理解 workflow core 的最小配置示例，不是主 demo 配置。

## 测试结构

后端测试按职责拆分：

- `tests/unit/workflow/`：registry、factory、engine 和架构约束。
- `tests/unit/nodes/`：schema linking、retrieval、mock LLM、生成节点、策略反思、SQL 修复、reasoning rewrite、HITL。
- `tests/unit/sql/`：SQL 方言和校验。
- `tests/unit/prompts/`：prompt pruning 摘要和模板渲染。
- `tests/integration/`：API、CLI、demo 场景、策略反思闭环、修复终止路径和数据库初始化。

前端测试覆盖：

- `frontend/src/App.test.tsx`：主交互、运行配置、Trace 展开、错误处理、SQL 编辑执行。
- `frontend/src/adapters/queryRunAdapter.test.ts`：API 响应到 UI view model 的适配。

维护规范见 [文档维护规范](文档维护规范.md)。
