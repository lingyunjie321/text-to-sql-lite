# 数据库与方言扩展

本文档面向二次开发者，说明如何连接服务型数据库（PostgreSQL/MySQL）以及如何扩展项目尚未内置的数据库类型（如 SQL Server、Oracle、DuckDB）。普通使用者用默认 SQLite 即可，不需要阅读本文档。

## 方言与 driver 映射

项目当前内置三套方言，`DialectName` 定义在 `src/text_to_sql_demo/sql/dialect.py`：

| driver（连接） | target_dialect（校验/生成） | execution_dialect（执行） | SQLGlot dialect | SQLAlchemy driver |
| --- | --- | --- | --- | --- |
| `sqlite` | `sqlite` | `sqlite` | `sqlite` | `sqlite` |
| `postgresql` | `postgres` | `postgres` | `postgres` | `psycopg2` |
| `mysql` | `mysql` | `mysql` | `mysql` | `pymysql` |

**关键约束**：校验方言、生成方言和执行方言必须一致。`ExecuteSQLNode` 会检查 `execution_dialect == validated_sql_dialect`，不一致直接返回 `dialect_error`，避免把按一种方言校验的 SQL 发到另一种数据库。

## 连接 PostgreSQL

### 1. 安装 driver

```bash
pip install -e ".[dev,db]"
```

`pyproject.toml` 的 `[project.optional-dependencies]` 的 `db` 段已包含 `psycopg2-binary`。如果用其他 driver（如 `asyncpg`），需自行安装并确认 SQLAlchemy 能识别。

### 2. 配置 workflow.yaml

在 `database.connections` 下新增连接，使用结构化字段（密码只放环境变量）：

```yaml
database:
  default: demo_postgres
  connections:
    demo_postgres:
      driver: postgresql
      host: localhost
      port: 5432
      database_name: text_to_sql_demo
      username: readonly_user
      password_env: TEXT_TO_SQL_DB_PASSWORD   # 密码只放环境变量，不要明文写进 YAML
      query:
        sslmode: prefer
      read_only: true
```

### 3. 同步方言配置

`workflow.yaml` 中所有方言相关节点必须保持 `postgres`：

```yaml
dialect:
  name: postgres
  allow_transpile: false
  target_dialect: postgres

nodes:
  sql_generation:
    target_dialect: postgres
  sql_validation:
    target_dialect: postgres
    render_dialect: postgres
  sql_execution:
    execution_dialect: postgres
```

### 4. 配置环境变量

```bash
# .env.local
TEXT_TO_SQL_DB_PASSWORD=your-readonly-password
```

也可以用 URL 环境变量覆盖（优先级最高）：

```bash
DEMO_DATABASE_URL=postgresql+psycopg2://readonly_user:password@localhost:5432/text_to_sql_demo
```

### 5. 权限要求

业务目标库的账号**必须只读**。项目层的只读安全依赖 SQLGlot AST 解析和执行前校验，不等同于生产级数据库权限隔离。建议在数据库侧也授予只读权限：

```sql
GRANT SELECT ON ALL TABLES IN SCHEMA public TO readonly_user;
ALTER DEFAULT PRIVILEGES IN SCHEMA public GRANT SELECT TO readonly_user;
```

## 连接 MySQL

步骤与 PostgreSQL 一致，driver 改为 `mysql`，driver 包为 `pymysql`：

```yaml
database:
  default: demo_mysql
  connections:
    demo_mysql:
      driver: mysql
      host: localhost
      port: 3306
      database_name: text_to_sql_demo
      username: readonly_user
      password_env: TEXT_TO_SQL_DB_PASSWORD
      read_only: true
```

方言配置全部改为 `mysql`。URL 形式：`mysql+pymysql://readonly_user:password@localhost:3306/text_to_sql_demo`。

## 使用运行时配置（不改 workflow.yaml）

如果只是临时连一个数据库，不必改 `workflow.yaml`，通过运行时配置 API 创建临时会话即可：

```bash
curl -X POST http://127.0.0.1:8000/api/v1/runtime/configs \
  -H 'Content-Type: application/json' \
  -d '{
    "database": {
      "mode": "custom",
      "config": {
        "driver": "postgresql",
        "host": "localhost", "port": 5432,
        "database_name": "analytics", "username": "readonly_user",
        "password": "runtime only",
        "target_dialect": "postgres",
        "display_name": "analytics"
      }
    },
    "models": {
      "light": {"mode": "preset", "preset_id": "light"},
      "strong": {"mode": "preset", "preset_id": "strong"}
    }
  }'
```

返回的 `runtime_config_id` 可用于 `/api/v1/query`、`/api/v1/schema`、`/api/v1/sql/execute`。运行时配置只存内存，服务重启失效。详见 [运行时配置](运行时配置.md)。

## 扩展新的数据库类型

以 SQL Server 为例，说明扩展一个新数据库类型需要改动的所有点。

### 改动点清单

| 序号 | 文件 | 改动 |
| --- | --- | --- |
| 1 | `pyproject.toml` | 在 `[project.optional-dependencies]` 的 `db` 段加 `pyodbc` 或 `sqlalchemy-pytds` |
| 2 | `src/text_to_sql_demo/sql/dialect.py` | `DialectName` Literal 加 `"sqlserver"`；`SUPPORTED_DIALECTS` 加 `"sqlserver"` |
| 3 | `src/text_to_sql_demo/config/models.py` | `DatabaseConnectionConfig.driver` Literal 加 `"sqlserver"` |
| 4 | `src/text_to_sql_demo/runtime/models.py` | `RuntimeDriver` Literal 加 `"sqlserver"`；`RuntimeDatabaseConfig.target_dialect` 扩展 |
| 5 | `src/text_to_sql_demo/nodes/sql_execution.py` | `SUPPORTED_EXECUTION_DIALECTS` 加 `"sqlserver"` |
| 6 | SQLGlot 确认 | SQLGlot 支持 `tsql` 方言，确认 `parse_one(sql, dialect="tsql")` 可用 |
| 7 | `configs/dialect_patterns.yaml` | 补充 SQL Server 业务范式（如 `TOP N` 而非 `LIMIT`） |
| 8 | `configs/examples.yaml` | 补充 SQL Server 方言示例 |
| 9 | 测试 | 补 `tests/unit/sql/test_dialect.py` 的 SQL Server 解析/渲染；补 `tests/unit/runtime/test_resolver.py` 的 driver→dialect 映射 |
| 10 | 文档 | 更新本表、[运行时配置](运行时配置.md) 的 driver 映射、README 限制说明 |

### 不要改动的部分

以下部分**不需要**改动，这是注册式架构的约束：

- `WorkflowEngine` — 不感知具体数据库类型
- `NodeFactory` / `NodeRegistry` — 不按数据库类型分发
- `BaseNode` 接口 — 不变
- 节点流转逻辑 — 由 `workflow.yaml` 的 edges 决定
- `PromptBuilder` — 方言无关

### driver → dialect → execution 映射对照

扩展时需要维护三处一致性：

```text
RuntimeDatabaseConfig.driver      ──►  "sqlserver"
RuntimeDatabaseConfig.target_dialect ──►  需要在 DialectName 加 "sqlserver"
ExecuteSQLNode execution_dialect  ──►  需要在 SUPPORTED_EXECUTION_DIALECTS 加 "sqlserver"
SQLGlot dialect                   ──►  SQLGlot 用 "tsql"，需要确认映射
```

`RuntimeConfigResolver` 会把 driver 映射到 target_dialect，扩展时确认 `src/text_to_sql_demo/runtime/resolver.py` 的映射逻辑覆盖新 driver。

### 测试要求

- `tests/unit/sql/test_dialect.py`：新增方言的 parse/normalize/transpile 测试。
- `tests/unit/runtime/test_resolver.py`：新增 driver → dialect 解析测试。
- `tests/unit/runtime/test_models.py`：新 driver 的配置校验测试。
- 不要在测试中连真实数据库；用 Mock 或 SQLite fallback 保持测试确定性。

## 安全约束

- 密码只放环境变量或运行时配置（内存），不写进 `workflow.yaml` 明文。
- 业务目标库默认只读，`read_only: true`。
- SQL 只读安全依赖 SQLGlot AST 解析（拒绝 insert/update/delete/drop/create/alter/command）和执行前校验，**不等同于**生产级数据库权限隔离。
- 建议在数据库侧也授予只读权限作为纵深防御。
- `metadata.db` 是项目内部运行记录库，`database.sqlite_discovery` 默认排除它，不作为业务查询数据源暴露。

## 边界

- 当前不宣称生产级数据库权限隔离。
- PostgreSQL/MySQL 真实连接依赖本地安装驱动、只读账号和真实连接环境。
- 扩展新数据库类型需要同时维护 driver、runtime model、API 映射、SQLGlot 方言和测试矩阵，但不要改动工作流引擎或节点流转逻辑。

维护规范见 [文档维护规范](文档维护规范.md)。
